### What is Devops?

- It is a culture or practice that we are going to follow that improves organization delivering power

- if my company takes 10 days to implement a feature its not a good practice, or if it is a bug if its take 10 days its not good practice

- if we can fix that bug or improving delievering power in shorter time is called Devops

## DevOps is a process of improving the application delivery by ensuring that there is a proper automation,quality and continous monitoring and testing

### Why Devops?

- Before Devops, Devolopers used to write the code and the end customers used to consume it in between there are multiple ppl involved in it. 
- **System admin** - Used to create the centralized server to deploy all the applications developed in the company
- **Build Release Engineers** - They used to deploy the application in the server
- **Server Admins** - they used to create the application server in the centralized server

- Whole process is time consuming and manual process then the devops evolved.
